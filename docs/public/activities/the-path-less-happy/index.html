The Path Less Happy

Objective: Understand that there is more than one path through the system and that different paths can be represented as swim lanes in an event storming diagram.

Prep:

  • Slide 1: Sample requirements (maybe on the wall?) and event storming. Use “Bike From Mike” example.
  • Slide 2: Additional requirement 1 not on the happy path (delete, update, ???)
  • Yellow postits and pens.

Howto:

  1. “Take a look at these requirements and the event storming. That’s a ‘happy path’.”

  2. “Now … spend two minutes on a second event stream which shows what happens when (deletion, etc.)”

  3. Instructor shows slide with happy path and two alternative flows.
