Workshop: Delivering Effective Microservices

  • Getting System Boundaries Right: Where does one microservice end and the next one begin, and why? How to get coupling & cohesion right first time using event storming, strategic Domain Driven Design, CQRS (Command Query Responsibility Segregation) and event sourcing.

  • Hands-on implementation, end-to-end. We will build a working system during the workshop, from scratch. Expect a working REST API with business logic and a minimalist user interface. Participants will have access to the source code after the event.

  • Leveraging legacy code investments. How to use event storming to understand legacy code and integrate it with new features and products. You will get a blueprint for doing legacy integrations in a plannable, predictable, least side effect prone way which fits your environment, project timelines and culture.

  • Project planning! Moving to CQRS and event sourced microservices drastically reduces coordination overhead and the need for regression testing. We’ll share some of the tricks we learned – Adaptech builds exclusively event-sourced CQRS systems and we have been doing it for a long time.
