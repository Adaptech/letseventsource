Context Mapping TODO

Prep: Slides with Kanban board requirements. Webeventstorming.com for same. Slides with Restaurant requirements. Webeventstorming.com for it. Howto: Understand context mapping: (4). Show the Restaurant Kitchen Show the Kanban board. Discuss for 2 minutes: How would the events for the kitchen be mapped to the events on the Kanban board? Ask a table group to present what they found. Show other examples which could be mapped to a TODO list in this way, then give table groups two minutes to come up with their own examples. »

Recipe Reviews - No tech jargon

Requires an envelope with events in random order - some are business, some are technical. One per table group. Use the EMD in recipe review example. Print it with https://gogs.adaptechsolutions.ca/robert.reppel/emd-pdf. Howto: “Take two minutes: Read the requirements. Then take a look at the events in the envelope. Try to arrange them in the right order.” Participants will run into difficulties putting the technical events in sequence and will start discussing implementation details. »

The Human Eventstorming

Print this PDF: Human Eventstorming Ask for volunteers from the audience. Have each of them grab one of the printed sheets with events. “Now arrange yourselves in the order in which the events occur.” »

The Path Less Happy

Objective: Understand that there is more than one path through the system and that different paths can be represented as swim lanes in an event storming diagram. Prep: Slide 1: Sample requirements (maybe on the wall?) and event storming. Use “Bike From Mike” example. Slide 2: Additional requirement 1 not on the happy path (delete, update, ???) Yellow postits and pens. Howto: “Take a look at these requirements and the event storming. »

Word Cloud

Prep: Print cards with words and phrases from three different bounded contexts, one set per table group. Put them in envelopes labeled “Word Cloud”. Tell participants to not open the envelopes until instructed to do so. Howto “Open the ‘Word Cloud” envelope. Group the cards in it - which ones belong together?” Then ask three table groups to tell a story about one of the bounded contexts without mentioning vocabulary of the other ones. »

Event Sourced Project Management

Prep Do (or provide result of, on webeventstorming.com) shopping cart event storming Slide: Shopping cart example bounded contexts. Slide: Blank Kanban board - TODO-Doing-Done Slide: Explain “Upstream” and “Downstream” teams/bounded contexts. Howto Say Do “Here are the results of the shopping cart exercise.” Show the shopping cart event storming on webeventstorming.com. “What bounded contexts do you see?” After the audience has identified contexts, show a slide with the bounded contexts. »
